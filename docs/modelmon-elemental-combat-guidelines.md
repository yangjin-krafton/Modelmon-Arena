# 모델몬 전투 원소 지침

## 목적

이 문서는 `모델몬` 전투 시스템에서 사용할 AI 원소 체계와 상태이상 체계를 고정하는 기준 문서입니다.

현재 목표는 세 가지입니다.

- 전투용 AI 원소를 8~12개 수준으로 고정한다.
- 원소 상성을 원작 포켓몬 타입표가 아니라 `AI 개념 충돌` 기준으로 다시 설계한다.
- 포켓몬 1기 기술 165개를 이 상성표에 맞는 스킬 라이브러리로 다시 읽는다.

## 확정 전투 원소

이번 라운드에서는 전투 원소를 `12개`로 고정합니다.

| 전투 원소 | 역할 | 이전 임시 분류 정리 |
| --- | --- | --- |
| 대화 | 지시, 설득, 문장 기반 압박 | 대화 유지 |
| 추론 | 단계적 사고, 계획, 판정 | 추론 유지 |
| 생성 | 샘플링, 변주, 고열 출력 | 생성 유지 |
| 검색 | 근거 회수, 랭킹, 노출 | 검색 유지 |
| 코드 | 실행, 수정, 결정적 처리 | 코드 유지 |
| 에이전트 | 도구 호출, 워크플로, 행동 전환 | `도구사용` 통합 |
| 멀티모달 | 이미지, 문서, 음성, 복합 입력 | 멀티모달 유지 |
| 메모리 | 문맥 저장, 회복, 흡수, 누적 | `기억` 통합 |
| 정렬 | 정책, 안전장치, 차단, 정지 | 정렬 유지 |
| 시스템 | 속도, 배포, 처리량, 인프라 | `실시간`, `인프라` 통합 |
| 학습 | 파인튜닝, 적응, 성장, 튜닝 | `프런티어`를 학습 축으로 정리 |
| 오염 | 환각, 독성, 혼선, 데이터 오염 | `환각`을 전투 원소로 재정의 |

## 상태군 재배치

상태이상은 원작 포켓몬의 독, 화상, 마비 같은 이름을 그대로 쓰지 않고 `AI 전투 상태군`으로 재배치합니다.

| 상태군 | 상태 원소 | 의미 |
| --- | --- | --- |
| 과열 | 생성 | 고출력으로 인한 과부하, 불안정 |
| 오염 | 오염 | 독성 데이터, 환각 누적 |
| 지연 | 시스템 | 속도 저하, 응답 끊김, 경직 |
| 정지 | 정렬 | 수면, 동결, 안전 중단 |
| 혼선 | 오염 | 혼란, 방향 상실, 무작위화 |
| 노출 | 검색 | 방어 약화, 정밀 노출, 취약점 드러남 |
| 속박 | 에이전트 | 기술 봉쇄, 행동 잠금, 구속 |
| 보호 | 정렬 | 장벽, 반사, 규칙 보호 |
| 회복 | 메모리 | 흡수, 재생, 문맥 회복 |
| 증폭 | 학습 | 자가 강화, 집중, 성장 |
| 반동 | 시스템 | 재충전, 자폭, 비용 역류 |
| 전환 | 에이전트 | 교체, 복제, 변환, 위치 이동 |

## 165개 스킬 점검 기준

포켓몬 1기 기술 165개는 아래 기준으로 다시 읽습니다.

- 스킬 수와 원본 기술 정보는 유지한다.
- `원본 타입`은 참고 정보로 남기되, 실제 전투 원소는 `모델몬 전투 원소` 기준으로 다시 매긴다.
- 이전 임시 원소인 `실시간`, `인프라`, `도구사용`, `기억`, `프런티어`, `환각`은 정규 원소로 통합한다.
- 각 기술은 `전투 원소 1개`와 `상태군 1개`를 가진다.
- 상태군이 없는 기술은 `없음`으로 둔다.
- 상성 계산은 앞으로 `전투 원소`만 사용하고, `AI 개념`과 `키워드`는 설명용 축으로 남긴다.
