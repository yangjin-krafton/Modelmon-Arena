# Modelmon Roguelike Adventure System Plan

## 목적

이 문서는 현재 구현 중인 `전투 1회 단위 플레이`를 확장해,  
`포켓로그`에 가까운 `런 기반 로그라이크 모험 구조`로 게임 전체를 시스템화하기 위한 기획서다.

핵심 목표는 아래 4가지다.

- 전투를 고립된 미니게임이 아니라 `런(run)` 안의 한 노드로 재정의한다.
- 야생 모델몬, 트레이너, 관장, 보스, 상점, 휴식, 이벤트를 `맵 노드 시스템`으로 통합한다.
- 매 전투 종료 후 보상과 선택이 누적되어 `빌드가 달라지는 로그라이크 감각`을 만든다.
- 현재 코드베이스의 전투 시스템을 최대한 유지하면서 바깥 루프를 점진적으로 붙일 수 있게 한다.

---

## 게임 한 판의 구조

한 번의 플레이는 `스타터 선택 -> 지역 진입 -> 노드 이동 반복 -> 지역 보스 클리어 -> 다음 지역 진입 -> 최종 보스 or 패배` 흐름으로 진행된다.

기본 루프는 아래와 같다.

1. 스타터 1마리를 선택하고 런을 시작한다.
2. 분기형 맵에서 다음 노드를 고른다.
3. 노드 종류에 따라 전투, 상점, 회복, 이벤트, 포획, 강화 중 하나가 발생한다.
4. 결과에 따라 팀, 재화, 스킬, 버프, 소모품, 영구 해금 상태가 변한다.
5. 지역 끝에서는 `관장` 또는 `지역 보스`와 싸운다.
6. 여러 지역을 돌파하면 최종 보스에 도달한다.
7. 런 종료 후 결과를 정산하고 메타 진행도를 반영한다.

이 구조에서 중요한 것은 `전투 승리`보다 `경로 선택과 리소스 관리`다.

---

## 디자인 원칙

### 1. 전투는 유지하고 바깥 루프를 확장한다

현재 `src/system/core/battle-engine.js`, `src/system/ui/battle.js`는 이미 단일 전투를 처리한다.  
따라서 첫 확장은 전투를 뜯어고치는 것이 아니라, 전투 전후에 붙는 `모험 상태 머신`을 만드는 쪽이 맞다.

### 2. 매 노드는 짧고 선택은 무겁게 만든다

포켓로그식 플레이 감각은 긴 탐험 애니메이션보다,
`짧은 노드 진행 + 자주 오는 선택 + 빌드 누적`에서 나온다.

### 3. 포획보다 편성 변화가 핵심이다

야생 조우는 단순 도감 채우기보다, 현재 런에서 팀을 교체하거나 보충하는 `전술 선택`이어야 한다.

### 4. 메타 진행은 런의 긴장감을 해치지 않게 제한한다

메타 해금은 `새 스타터, 새 이벤트, 새 상점 상품, 새 특성 풀` 중심으로 두고,
초반부터 수치를 과도하게 영구 강화하는 구조는 피한다.

## 런 구조 제안

### 지역 수

- 기본 1런은 `3개 지역 + 최종 구간 1개`를 권장
- 총 길이는 `15~22노드` 수준을 목표로 한다
- 한 지역은 `일반 노드 4~6개 + 중간 강적 1개 + 보스 1개` 구조가 적절하다

### 지역 예시

- `Starter Zone`
  - 튜토리얼 성격
  - 야생, 약한 트레이너, 기본 상점만 등장
- `Protocol City`
  - 정렬/규칙/보안 계열 적 증가
  - 상태이상, 방어형 적 비율 상승
- `Inference Wilds`
  - 생성/추론/멀티모달 계열 야생 비율 상승
  - 희귀 포획 확률 증가
- `Frontier Core`
  - 엘리트, 보스, 고난도 이벤트 집중
  - 최종 보스 등장

## 노드 타입 설계

모든 맵 노드는 아래 공통 필드를 가진다.

- `nodeId`
- `regionId`
- `floorIndex`
- `nodeType`
- `dangerTier`
- `rewardSeed`
- `encounterSeed`
- `cleared`

### 1. 야생 조우

가장 자주 등장하는 기본 노드다.

- 적 1마리 또는 2마리 편성
- 승리 시 포획 기회, 재화, 경험치, 드랍 중 일부 제공
- 낮은 위험, 높은 유연성

주요 목적:

- 팀 보충
- 도감 조우 확장
- 약한 리소스 회복 없이도 돌파 가능한 기본 전투

### 2. 트레이너 배틀

야생보다 강하고 보상도 큰 표준 전투 노드다.

- 적 2~3마리 편성
- 명확한 역할 조합을 가진 팀 사용
- 승리 시 재화와 업그레이드 보상이 큼

주요 목적:

- 중간 난이도 체크
- 빌드 파워 상승의 핵심 보상 제공

### 3. 엘리트

런의 중간 점검 노드다.

- 강한 시너지 팀 구성
- 특수 룰, 날씨, 필드 효과, 시작 버프 등 부여 가능
- 승리 시 희귀 업그레이드, 특성, 고급 상점권 지급

### 4. 관장

지역 테마를 대표하는 보스 노드다.

- 특정 AI 개념 또는 전투 스타일에 특화
- 예: `정렬 관장`, `추론 관장`, `생성 관장`
- 지역 클리어 보상과 다음 지역 진입 권한 제공

### 5. 최종 보스

런 종착점이다.

- 다단계 페이즈 또는 특수 패턴 보유
- 지금까지 만든 빌드의 종합 테스트
- 보상은 메타 해금 중심

### 6. 상점

전투 외 리소스 전환 노드다.

판매 항목 예시:

- 회복 패치
- PP 충전
- 일회성 전투 아이템
- 스킬 교체권
- 팀 슬롯 확장
- 희귀 포획 모듈
- 패시브 버프

### 7. 휴식

안정성을 주는 대신 파워 상승량은 낮은 노드다.

선택지 예시:

- HP 회복
- PP 회복
- 상태이상 제거
- 소량 경험치 훈련
- 팀원 1명 친밀도 상승

### 8. 랜덤 이벤트

텍스트 선택형 노드다.

예시:

- 수상한 연구소: 강한 보상 대신 디버프 위험
- 버려진 서버실: 희귀 모델몬 조우
- 프롬프트 딜러: 랜덤 스킬 교환
- 패치 충돌: 현재 버프를 잃고 재화를 얻음

### 9. 포획 특화 노드

희귀 모델몬 획득을 위한 별도 노드다.

- 일반 야생보다 포획 확률이 높음
- 전투 없이 계약, 교환, 선택형 영입으로도 처리 가능

## 맵 진행 구조

### 기본 형태

- `Slay the Spire`식 분기 맵을 사용한다
- 한 줄로 길게 가는 방식보다 `2~4개의 경로 선택`이 보이는 구조가 적합하다

### 노드 분포 규칙 예시

- 초반 3노드 내 상점 1회 이하
- 보스 직전에는 휴식 또는 상점 중 하나 배치
- 연속 엘리트는 금지
- 관장 직전 분기에서는 위험도 높은 노드에 보상을 크게 준다

### 지역별 노드 풀 예시

- 초반: 야생, 약한 트레이너, 휴식 중심
- 중반: 상점, 엘리트, 이벤트 비율 증가
- 후반: 강한 트레이너, 엘리트, 관장, 희귀 상점 중심

---

## 전투 난이도 스케일링

현재 전투는 스타터 1마리 기준에 가깝다.  
로그라이크 구조에서는 적과 플레이어 모두 `런 단계`에 따라 강해져야 한다.

### 난이도 축

- `regionRank`
- `nodeDanger`
- `winStreak`
- `teamSize`
- `starterPowerScore`

### 적 생성 규칙

- 야생: 현재 지역 기준 평균 레벨 ±1
- 트레이너: 평균 레벨 +1~2
- 엘리트: 평균 레벨 +2~4, 시너지 조합 포함
- 관장/보스: 평균 레벨 +4 이상, 전용 패턴 포함

### 추가 난이도 장치

- 특정 전투에서 시작 버프 적용
- 필드 효과 적용
- 특정 타입 출현율 강화
- 교체 제한, 회복 제한, 상태이상 강화

## 플레이어 팀 시스템

### 팀 크기

- 초기 팀은 `1마리`
- 런 중 최대 `3마리` 또는 `4마리`까지 확장 권장
- MVP 기준은 `최대 3마리`가 현실적이다

### 영입 방식

- 야생 포획
- 이벤트 영입
- 상점 계약
- 보스 격파 후 특수 동료 획득

### 팀 운영 규칙

- 전투마다 선봉 1마리 선택
- 교체는 허용하되 비용 또는 턴 손해를 준다
- HP/PP는 런 전체에서 부분 유지

이렇게 해야 `다음 전투를 위해 누구를 아끼는가`라는 전략이 생긴다.

---

## 보상 시스템

전투 후 보상은 항상 `즉시 강해지는 선택`이 느껴져야 한다.

### 기본 보상 축

- 경험치
- 크레딧
- 포획 기회
- 스킬 선택
- 패시브 버프
- 소모품
- 진화 조건 진척

### 전투 후 선택지 예시

승리 후 3택 보상:

- 스킬 강화
- 재화 획득
- 회복 아이템 획득

엘리트 승리 후 3택 보상:

- 희귀 패시브
- 팀 슬롯 확장
- 고급 상점 할인권

### 보상 설계 원칙

- 항상 `안정성`, `공격성`, `탐욕` 중 하나를 고르게 한다
- 같은 런에서도 빌드 방향이 갈리도록 상충 보상을 섞는다

## 성장 시스템

### 런 내부 성장

- 레벨업
- 스킬 습득/교체
- 진화
- 패시브 획득
- 장착형 모듈 획득

### 메타 성장

- 새 스타터 해금
- 새 지역 해금
- 새 이벤트 해금
- 새 상점 상품 풀 해금
- 희귀 야생 조우 풀 확장
- 관장/보스 도감 기록

메타 성장은 `새 선택지 해금` 위주로 설계하고,  
기본 공격력 같은 단순 영구 수치 상승은 최소화한다.

---

## 진화와 빌드 변화

현재 프로젝트의 도감/진화 기반 자산을 살리려면 런 안에서 `진화 타이밍`이 중요한 선택이 되어야 한다.

### 진화 조건 예시

- 레벨 도달
- 특정 타입 적 격파 횟수
- 특정 이벤트 통과
- 특정 아이템 사용
- 특정 패시브 보유

### 진화 설계 원칙

- 진화는 단순 스탯 상승이 아니라 `역할 변화`를 만들어야 한다
- 예: 속공형에서 제어형으로, 범용형에서 생성 특화형으로 이동

---

## 상점 시스템 상세

상점은 단순 회복소가 아니라 `재화를 빌드로 바꾸는 장소`여야 한다.

### 화폐 제안

- 기본 화폐: `크레딧`
- 희귀 화폐: `코어 샤드`

### 상품 카테고리

- 회복
- 소모품
- 패시브 모듈
- 포획 보조
- 스킬 관련 상품
- 팀 확장 상품

### 상점 변주

- 일반 상점
- 블랙마켓
- 연구소 거래소
- 관장 전용 상점

상점 종류를 나누면 같은 `shop` 노드도 체감이 크게 달라진다.

---

## 이벤트 시스템 상세

이벤트는 전투를 쉬게 만드는 장치가 아니라,
`확률과 리스크를 감수하는 선택 노드`가 되어야 한다.

### 이벤트 선택지 구조

- 안전 선택: 작은 이득
- 모험 선택: 큰 이득, 확률 리스크
- 빌드 선택: 특정 타입/전략에만 유리

### 이벤트 예시

- `불안정한 체크포인트`
  - 전원 회복
  - 하지만 다음 2전투 동안 최대 HP 감소
- `폐기된 실험체`
  - 랜덤 희귀 모델몬 영입
  - 하지만 팀 전체 디버프 가능
- `튜닝 콘솔`
  - 스킬 하나를 강화
  - 다른 스킬 하나는 제거

## 적 분류 체계

로그라이크 시스템에서는 적을 `개별 몬스터`보다 `조우 패턴` 기준으로 묶는 것이 중요하다.

### 적 등급

- 일반 야생
- 희귀 야생
- 트레이너
- 엘리트 트레이너
- 관장
- 보스
- 특수 이벤트 적

### 조우 패턴 태그 예시

- `burst`
- `stall`
- `summon`
- `status`
- `swap`
- `setup`
- `risk-reward`

이 태그를 사용하면 나중에 적 생성기가 `전투 스타일` 중심으로 조합을 만들 수 있다.

## 데이터 구조 제안

현재 저장 구조는 도감/스타터 해금 위주다.  
로그라이크 확장을 위해 아래 상태가 추가되어야 한다.

### `runState` 제안

```js
{
  active: true,
  seed: 123456,
  regionIndex: 0,
  floorIndex: 2,
  nodeId: "r0-f2-b",
  credits: 120,
  shards: 1,
  team: [
    {
      monId: "001",
      lv: 7,
      hp: 24,
      maxHp: 28,
      ppState: { "tackle": 18 },
      passiveIds: ["starter-bonus"],
      itemId: null
    }
  ],
  reserve: [],
  inventory: ["potion"],
  map: [],
  history: {
    visitedNodes: [],
    capturedIds: [],
    defeatedBossIds: []
  }
}
```

### 저장 분리 원칙

- `metaSave`: 영구 해금 데이터
- `runSave`: 진행 중인 런 데이터

이 둘을 분리해야 죽었을 때 런만 초기화하고 메타는 유지할 수 있다.

## 현재 코드와의 연결 포인트

## 콘텐츠 우선순위

처음부터 모든 노드를 다 넣으면 시스템 복잡도만 급증한다.  
따라서 아래 순서로 여는 것이 맞다.

### 1차 MVP

- 스타터 선택
- 분기형 맵
- 야생 노드
- 트레이너 노드
- 상점 노드
- 휴식 노드
- 지역 보스 1종
- 런 저장/이어하기

### 2차 확장

- 엘리트 노드
- 랜덤 이벤트
- 포획 특화 노드
- 팀 3마리 운영
- 진화 분기 강화

### 3차 확장

- 관장 다변화
- 지역별 전용 규칙
- 희귀 상점
- 최종 보스 다단계 패턴
- 메타 해금 트리

---

## 밸런스 원칙

### 초반

- 플레이어가 쉽게 팀을 잃지 않게 한다
- 대신 자원 선택에서 실수가 누적되게 만든다

### 중반

- 빌드 차이가 체감되기 시작해야 한다
- 상점, 이벤트, 엘리트 선택이 런 성패를 가른다

### 후반

- 단순 레벨보다 시너지와 상태 관리가 중요해야 한다
- 보스는 수치가 아니라 패턴으로 압박해야 한다

---

## 추천 구현 순서

1. `runState`와 저장 구조를 분리한다.
2. `맵 생성기`와 `노드 이동 화면`을 만든다.
3. 전투 시작 함수를 `encounterData` 기반으로 바꾼다.
4. 전투 후 `보상 선택 화면`을 붙인다.
5. 상점/휴식 노드를 추가한다.
6. 지역 보스와 런 완료/실패 처리를 넣는다.
7. 이후 이벤트와 메타 해금을 확장한다.

## 최종 방향 정리

Modelmon Arena의 다음 단계는 `더 화려한 전투`보다 `전투를 감싸는 반복 가능한 모험 구조`를 만드는 것이다.

가장 중요한 시스템 축은 아래 다섯 가지다.

- `런 상태`
- `분기 맵`
- `노드 타입`
- `전투 후 보상`
- `메타 해금`

이 다섯 축만 안정적으로 굴러가면,  
야생 모델몬, 트레이너, 관장, 보스, 상점, 이벤트는 개별 콘텐츠가 아니라 같은 로그라이크 틀 안에서 자연스럽게 늘릴 수 있다.
